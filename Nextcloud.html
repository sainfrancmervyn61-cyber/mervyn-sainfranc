Guide Nextcloud — Déploiement Complet

Documentation complète pour déployer une infrastructure Nextcloud avec Docker, Nginx, MariaDB, VPN WireGuard et intégration NAS Synology. Ports par défaut utilisés : 443 (HTTPS), 22 (SSH), 51820 (WireGuard).

📦 Version : Nextcloud 28
🐳 Conteneurs : Docker
🌐 Serveur Web : Nginx
🗄️ Base de données : MariaDB 10.6
📋
Introduction
Présentation du projet et objectifs

Ce guide vous permet de déployer une instance Nextcloud complète avec :

✓
Un serveur web Nginx avec certificat SSL auto-signé
✓
Une base de données MariaDB 10.6
✓
Une application Nextcloud 28 en conteneur Docker
✓
Un VPN WireGuard pour une connexion sécurisée avec le NAS
✓
Une intégration avec un NAS Synology via WebDAV
ℹ️ Ports utilisés :
  • 443 : HTTPS (accès web sécurisé)
  • 22 : SSH (administration)
  • 51820 : WireGuard (VPN)

Ces ports peuvent être modifiés selon vos besoins, mais ce guide utilise les valeurs par défaut pour simplifier la configuration.

Prérequis
Conditions nécessaires avant l'installation

Matériel et logiciel

✓
Un serveur sous Linux (Ubuntu 22.04 LTS ou Debian 11 recommandé)
✓
Docker et Docker Compose installés
✓
Accès root ou sudo
✓
Minimum 4 Go de RAM et 20 Go de disque disponibles
✓
Un NAS Synology accessible sur le réseau local
✓
Un nom de domaine ou une IP publique pour l'accès externe (optionnel)

Ports réseau

Les ports suivants doivent être disponibles sur votre serveur :

Service Port Protocole Description
Nextcloud (HTTPS) 443 TCP Accès web sécurisé
SSH 22 TCP Administration à distance
WireGuard VPN 51820 UDP Connexion VPN avec le NAS
⚠️ Important : Si ces ports sont déjà utilisés, vous devrez les modifier dans les configurations suivantes.

Installation avec Docker
Déploiement de Nextcloud en conteneurs

1. Nettoyage initial (si nécessaire)

Si une installation précédente existe, commencez par tout supprimer :

bash — Nettoyage complet
# Arrêter et supprimer les conteneurs existants
docker stop nextcloud-nginx nextcloud-app nextcloud-db
docker rm nextcloud-nginx nextcloud-app nextcloud-db

# Nettoyer les images et volumes inutilisés
docker system prune -af

# Supprimer les fichiers de configuration
sudo rm -rf /opt/nextcloud

2. Création de l'arborescence

bash — Créer les dossiers
sudo mkdir -p /opt/nextcloud/{app,data,db,nginx,ssl,config}

3. Génération du certificat SSL auto-signé

bash — Générer le certificat
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /opt/nextcloud/ssl/nextcloud.key \
  -out /opt/nextcloud/ssl/nextcloud.crt \
  -subj "/CN=nextcloud"
ℹ️ Note : Pour un certificat valide (sans avertissement dans le navigateur), utilisez Let's Encrypt avec Certbot.

4. Fichier docker-compose.yml

Créez le fichier de configuration Docker Compose :

bash — Créer le fichier
sudo nano /opt/nextcloud/docker-compose.yml
/opt/nextcloud/docker-compose.yml
version: '3.8'

services:

  # Base de données MariaDB
  db:
    image: mariadb:10.6
    container_name: nextcloud-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: CHANGE_ME_ROOT_PASSWORD
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: CHANGE_ME_DB_PASSWORD
    volumes:
      - /opt/nextcloud/db:/var/lib/mysql
    networks:
      - nextcloud_network

  # Application Nextcloud
  nextcloud:
    image: nextcloud:28-fpm
    container_name: nextcloud-app
    restart: always
    environment:
      MYSQL_HOST: db
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: CHANGE_ME_DB_PASSWORD
    volumes:
      - /opt/nextcloud/app:/var/www/html
      - /opt/nextcloud/data:/var/www/html/data
      - /opt/nextcloud/config:/var/www/html/config
    depends_on:
      - db
    networks:
      - nextcloud_network

  # Serveur Web Nginx
  nginx:
    image: nginx:latest
    container_name: nextcloud-nginx
    restart: always
    ports:
      - "443:443" # Port HTTPS par défaut
    volumes:
      - /opt/nextcloud/nginx/nextcloud.conf:/etc/nginx/conf.d/default.conf
      - /opt/nextcloud/ssl:/etc/nginx/ssl
      - /opt/nextcloud/app:/var/www/html
    depends_on:
      - nextcloud
    networks:
      - nextcloud_network

networks:
  nextcloud_network:
    driver: bridge
ℹ️ Note : Remplacez CHANGE_ME_ROOT_PASSWORD et CHANGE_ME_DB_PASSWORD par vos mots de passe réels.

5. Configuration Nginx

bash — Créer le fichier de config Nginx
sudo nano /opt/nextcloud/nginx/nextcloud.conf
/opt/nextcloud/nginx/nextcloud.conf
server {
    listen 443 ssl;
    server_name _;

    # Certificat SSL
    ssl_certificate /etc/nginx/ssl/nextcloud.crt;
    ssl_certificate_key /etc/nginx/ssl/nextcloud.key;

    # Protocoles SSL
    ssl_protocols TLSv1.2 TLSv1.3;

    # Racine du site
    root /var/www/html;
    index index.php;

    # Taille max des fichiers (2 Go)
    client_max_body_size 2048M;

    # Gestion des URLs
    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    # Gestion de PHP
    location ~ \.php$ {
        fastcgi_pass nextcloud-app:9000;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }

    # Désactiver le logging des requêtes PHP
    location ~* \.(?:css|js|jpe?g|gif|ico|png|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public";
        access_log off;
    }
}

6. Lancement des conteneurs

bash — Démarrer Nextcloud
cd /opt/nextcloud
docker compose up -d
✅ Accès à Nextcloud : https://VOTRE_IP_OU_DOMAINE

7. Configuration initiale de Nextcloud

Ouvrez votre navigateur et accédez à l'URL ci-dessus. Vous devriez voir l'assistant d'installation de Nextcloud :

1
Sélectionnez un nom d'utilisateur administrateur et un mot de passe
2
Renseignez les informations de la base de données :
  • Type de base de données : MySQL/MariaDB
  • Hôte : db (nom du service dans docker-compose)
  • Base de données : nextcloud
  • Utilisateur : nextcloud
  • Mot de passe : CHANGE_ME_DB_PASSWORD (celui défini dans docker-compose.yml)
3
Cliquez sur Terminer l'installation
⚠️ Important : Notez bien le nom d'utilisateur et le mot de passe administrateur. Ils seront nécessaires pour accéder à Nextcloud.

Configuration avancée
Optimisation et personnalisation

1. Configuration de base

Une fois l'installation terminée, connectez-vous à Nextcloud et effectuez les configurations suivantes :

✓
Allez dans Paramètres → Administration
✓
Vérifiez que le mode maintenance est désactivé
✓
Configurez l'email du serveur pour les notifications
✓
Activez les applications recommandées (OnlyOffice, Talk, etc.)

2. Configuration des domaines de confiance

Pour éviter les erreurs de sécurité, ajoutez votre domaine ou IP à la liste des domaines de confiance :

bash — Ajouter un domaine de confiance
docker exec -u www-data nextcloud-app php occ config:system:set trusted_domains 1 --value="VOTRE_IP_OU_DOMAINE"

3. Configuration du cron

Pour que les tâches planifiées (comme la suppression des fichiers temporaires) fonctionnent correctement :

bash — Configurer le cron
docker exec -u www-data nextcloud-app php occ background:cron
ℹ️ Note : Vous pouvez aussi configurer une tâche cron sur l'hôte pour exécuter cette commande toutes les 5 minutes : 
*/5 * * * * docker exec -u www-data nextcloud-app php cron.php

VPN WireGuard
Connexion sécurisée entre Nextcloud et le NAS

WireGuard est un VPN moderne, rapide et sécurisé. Nous allons l'utiliser pour connecter de manière sécurisée votre serveur Nextcloud à votre NAS Synology.

1. Installation de WireGuard

bash — Installer WireGuard
sudo apt update
sudo apt install wireguard -y

2. Génération des clés

Sur le serveur Nextcloud :

bash — Générer les clés pour Nextcloud
wg genkey | sudo tee /etc/wireguard/nextcloud_private.key | wg pubkey | sudo tee /etc/wireguard/nextcloud_public.key

Afficher les clés générées :

bash — Afficher les clés
# Clé privée Nextcloud (À PROTÉGER !)
sudo cat /etc/wireguard/nextcloud_private.key

# Clé publique Nextcloud
sudo cat /etc/wireguard/nextcloud_public.key
⚠️ Attention : La clé privée doit être stockée de manière sécurisée. Ne la partagez jamais en clair.

Sur le NAS Synology :

Connectez-vous en SSH au NAS et exécutez :

bash — Générer les clés pour le NAS
wg genkey | tee /etc/wireguard/nas_private.key | wg pubkey > /etc/wireguard/nas_public.key

3. Configuration WireGuard sur Nextcloud

bash — Configurer WireGuard
sudo nano /etc/wireguard/wg0.conf
/etc/wireguard/wg0.conf
[Interface]
Address = 10.8.0.1/24
PrivateKey = NEXTCLOUD_PRIVATE_KEY
ListenPort = 51820
PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = NAS_PUBLIC_KEY
AllowedIPs = 10.8.0.2/32

Activer WireGuard :

bash — Activer le VPN
sudo sysctl -w net.ipv4.ip_forward=1
sudo echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0

4. Configuration WireGuard sur le NAS

Sur le NAS Synology, créez un fichier de configuration WireGuard avec :

Configuration NAS (/etc/wireguard/wg0.conf)
[Interface]
Address = 10.8.0.2/24
PrivateKey = NAS_PRIVATE_KEY

[Peer]
PublicKey = NEXTCLOUD_PUBLIC_KEY
Endpoint = NEXTCLOUD_PUBLIC_IP:51820
AllowedIPs = 10.8.0.1/32, 192.168.1.0/24
PersistentKeepalive = 25

Activez WireGuard sur le NAS :

bash — Activer WireGuard sur le NAS
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0
ℹ️ Note : Remplacez NEXTCLOUD_PRIVATE_KEY, NAS_PUBLIC_KEY, etc. par vos valeurs réelles.

5. Test de la connexion VPN

Depuis Nextcloud :

bash — Tester la connexion vers le NAS
ping 10.8.0.2

Depuis le NAS :

bash — Tester la connexion vers Nextcloud
ping 10.8.0.1
✅ Validation : Si les pings fonctionnent, votre tunnel VPN est opérationnel.

Configuration du NAS Synology
Intégration avec Nextcloud via WebDAV

1. Activer WebDAV sur le NAS

Depuis l'interface DSM du NAS :

✓
Aller dans Panneau de configuration → Services de fichiers
✓
Activer WebDAV
✓
Activer HTTPS (port 5006 par défaut)
✓
Créer un dossier partagé Nextcloud (ex: /volume1/Nextcloud)
✓
Créer un utilisateur dédié (ex: nextcloud-user) avec accès au dossier

2. Connecter Nextcloud au NAS

Dans Nextcloud :

1
Aller dans Administration → Stockage externe
2
Ajouter un nouveau stockage externe
3
Sélectionner WebDAV comme type de stockage

Renseigner les informations suivantes :

Champ Valeur
URL https://10.8.0.2:5006/volume1/Nextcloud
Utilisateur nextcloud-user
Mot de passe XXXXXX
Options ✅ HTTPS
✅ Certificat auto-signé
✅ Validation : Vous devriez voir un point vert si la connexion est réussie.

3. Configuration des sauvegardes

Configurer une tâche de sauvegarde dans HyperBackup sur le NAS Synology :

✓
Dossiers à sauvegarder : /opt/nextcloud/app, /opt/nextcloud/data, /opt/nextcloud/db
✓
Destination : Un dossier dédié sur le NAS (ex: /volume1/Backups/Nextcloud)
✓
Planification : Quotidienne à 2h00
✓
Rétention : 30 jours

Configuration du Firewall UFW
Sécurisation du serveur Nextcloud

1. Installation de UFW

bash — Installer UFW
sudo apt update
sudo apt install ufw -y

2. Configuration des règles

bash — Autoriser les ports nécessaires
# Autoriser SSH (port 22)
sudo ufw allow 22/tcp

# Autoriser HTTPS (port 443)
sudo ufw allow 443/tcp

# Autoriser WireGuard (port 51820)
sudo ufw allow 51820/udp

3. Activation de UFW

bash — Activer le firewall
sudo ufw enable

# Vérifier les règles
sudo ufw status
✅ Résultat attendu : 
Status: active

To                         Action      From
--                         ------      ----
22/tcp                     ALLOW       Anywhere
443/tcp                    ALLOW       Anywhere
51820/udp                  ALLOW       Anywhere
22/tcp (v6)                ALLOW       Anywhere (v6)
443/tcp (v6)               ALLOW       Anywhere (v6)
51820/udp (v6)             ALLOW       Anywhere (v6)

Maintenance
Gestion et optimisation de Nextcloud

1. Mise à jour des conteneurs

bash — Mettre à jour Nextcloud
cd /opt/nextcloud
docker compose pull
docker compose up -d

2. Réparation de Nextcloud

bash — Réparer l'installation
docker exec -u www-data nextcloud-app php occ maintenance:repair

3. Ajout des index manquants

bash — Optimiser la base de données
docker exec -u www-data nextcloud-app php occ db:add-missing-indices

4. Exécution du cron

bash — Lancer les tâches planifiées
docker exec -u www-data nextcloud-app php cron.php

5. Vérification de l'état

bash — Vérifier l'état de Nextcloud
docker exec -u www-data nextcloud-app php occ status
docker exec -u www-data nextcloud-app php occ check

6. Sauvegardes manuelles

Pour effectuer une sauvegarde manuelle complète :

bash — Sauvegarder les données
# Sauvegarder la base de données
docker exec nextcloud-db mysqldump -u root -pCHANGE_ME_ROOT_PASSWORD nextcloud > /opt/nextcloud/backups/nextcloud-db-$(date +%Y-%m-%d).sql

# Sauvegarder les fichiers
sudo tar -czvf /opt/nextcloud/backups/nextcloud-files-$(date +%Y-%m-%d).tar.gz /opt/nextcloud/app /opt/nextcloud/data
