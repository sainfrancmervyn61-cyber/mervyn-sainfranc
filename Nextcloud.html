Guide Nextcloud — Déploiement Complet

Documentation complète pour déployer une infrastructure Nextcloud avec Docker, Nginx, MariaDB, VPN WireGuard et intégration NAS Synology. Ports par défaut utilisés : 443 (HTTPS), 22 (SSH), 51820 (WireGuard).

Version : Nextcloud 28
Conteneurs : Docker
Serveur Web : Nginx
Base de données : MariaDB 10.6
Aperçu du projet
Pourquoi ce guide a été créé

Ce guide vous permet de déployer une instance Nextcloud complète avec une infrastructure sécurisée et scalable. Il est conçu pour les administrateurs système qui souhaitent héberger leur propre solution de stockage cloud.

Les objectifs principaux sont :

Sécurité renforcée avec certificats SSL et firewall
Stockage cloud privé et contrôlé
Sauvegardes automatiques vers NAS
Collaboration et partage de fichiers
ℹ️ Public cible :
  • Administrateurs système et réseau
  • Techniciens DevOps
  • Étudiants en informatique
  • Toute personne souhaitant héberger son propre cloud

Démarrage rapide
Deployez Nextcloud en 10 minutes

1. Prérequis

✓
Un serveur sous Ubuntu 22.04 LTS ou Debian 11
✓
Docker et Docker Compose installés
✓
Accès root ou sudo
✓
Minimum 4 Go de RAM et 20 Go de disque
✓
Un NAS Synology accessible sur le réseau local

2. Installation de Docker

Si Docker n'est pas déjà installé :

bash — Installer Docker
sudo apt update
sudo apt install docker.io docker-compose -y
sudo systemctl enable docker
sudo systemctl start docker

3. Création de l'arborescence

bash — Créer les dossiers
sudo mkdir -p /opt/nextcloud/{app,data,db,nginx,ssl,config}

4. Génération du certificat SSL

bash — Générer le certificat
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /opt/nextcloud/ssl/nextcloud.key \
  -out /opt/nextcloud/ssl/nextcloud.crt \
  -subj "/CN=nextcloud"
ℹ️ Note : Pour un certificat valide (sans avertissement dans le navigateur), utilisez Let's Encrypt avec Certbot.

5. Fichier docker-compose.yml

bash — Créer le fichier
sudo nano /opt/nextcloud/docker-compose.yml
/opt/nextcloud/docker-compose.yml
version: '3.8'

services:
  # Base de données MariaDB
  db:
    image: mariadb:10.6
    container_name: nextcloud-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: CHANGE_ME_ROOT_PASSWORD
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: CHANGE_ME_DB_PASSWORD
    volumes:
      - /opt/nextcloud/db:/var/lib/mysql
    networks:
      - nextcloud_network

  # Application Nextcloud
  nextcloud:
    image: nextcloud:28-fpm
    container_name: nextcloud-app
    restart: always
    environment:
      MYSQL_HOST: db
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: CHANGE_ME_DB_PASSWORD
    volumes:
      - /opt/nextcloud/app:/var/www/html
      - /opt/nextcloud/data:/var/www/html/data
      - /opt/nextcloud/config:/var/www/html/config
    depends_on:
      - db
    networks:
      - nextcloud_network

  # Serveur Web Nginx
  nginx:
    image: nginx:latest
    container_name: nextcloud-nginx
    restart: always
    ports:
      - "443:443"
    volumes:
      - /opt/nextcloud/nginx/nextcloud.conf:/etc/nginx/conf.d/default.conf
      - /opt/nextcloud/ssl:/etc/nginx/ssl
      - /opt/nextcloud/app:/var/www/html
    depends_on:
      - nextcloud
    networks:
      - nextcloud_network

networks:
  nextcloud_network:
    driver: bridge
ℹ️ Note : Remplacez CHANGE_ME_ROOT_PASSWORD et CHANGE_ME_DB_PASSWORD par vos mots de passe réels.

6. Configuration Nginx

bash — Créer le fichier de config Nginx
sudo nano /opt/nextcloud/nginx/nextcloud.conf
/opt/nextcloud/nginx/nextcloud.conf
server {
    listen 443 ssl;
    server_name _;

    # Certificat SSL
    ssl_certificate /etc/nginx/ssl/nextcloud.crt;
    ssl_certificate_key /etc/nginx/ssl/nextcloud.key;

    # Protocoles SSL
    ssl_protocols TLSv1.2 TLSv1.3;

    # Racine du site
    root /var/www/html;
    index index.php;

    # Taille max des fichiers
    client_max_body_size 2048M;

    # Gestion des URLs
    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    # Gestion de PHP
    location ~ \.php$ {
        fastcgi_pass nextcloud-app:9000;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $fastcgi_path_info;
    }
}

bash — Démarrer Nextcloud

VPN WireGuard
Connexion sécurisée avec le NAS

WireGuard est un VPN moderne et performant. Nous allons l'utiliser pour connecter votre serveur Nextcloud à votre NAS Synology de manière sécurisée.

1. Installation de WireGuard

bash — Installer WireGuard
sudo apt update
sudo apt install wireguard -y

2. Génération des clés

Sur le serveur Nextcloud :

bash — Générer les clés
wg genkey | sudo tee /etc/wireguard/nextcloud_private.key | wg pubkey | sudo tee /etc/wireguard/nextcloud_public.key

Afficher les clés :

bash — Afficher les clés
# Clé privée (À PROTÉGER !)
sudo cat /etc/wireguard/nextcloud_private.key

# Clé publique
sudo cat /etc/wireguard/nextcloud_public.key
⚠️ Attention : La clé privée doit être stockée de manière sécurisée. Ne la partagez jamais.

Sur le NAS Synology :

bash — Générer les clés NAS
wg genkey | tee /etc/wireguard/nas_private.key | wg pubkey > /etc/wireguard/nas_public.key

3. Configuration WireGuard sur Nextcloud

bash — Configurer WireGuard
sudo nano /etc/wireguard/wg0.conf
/etc/wireguard/wg0.conf
[Interface]
Address = 10.8.0.1/24
PrivateKey = NEXTCLOUD_PRIVATE_KEY
ListenPort = 51820
PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = NAS_PUBLIC_KEY
AllowedIPs = 10.8.0.2/32

Activer WireGuard :

bash — Activer WireGuard
sudo sysctl -w net.ipv4.ip_forward=1
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0

4. Configuration WireGuard sur le NAS

Configuration NAS
[Interface]
Address = 10.8.0.2/24
PrivateKey = NAS_PRIVATE_KEY

[Peer]
PublicKey = NEXTCLOUD_PUBLIC_KEY
Endpoint = NEXTCLOUD_PUBLIC_IP:51820
AllowedIPs = 10.8.0.1/32, 192.168.1.0/24
PersistentKeepalive = 25

Activez WireGuard sur le NAS :

bash — Activer sur le NAS
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0
ℹ️ Note : Remplacez NEXTCLOUD_PRIVATE_KEY, NAS_PUBLIC_KEY, etc. par vos valeurs réelles.

5. Test de la connexion

Depuis Nextcloud :

bash — Tester vers le NAS
ping 10.8.0.2

Depuis le NAS :

bash — Tester vers Nextcloud
ping 10.8.0.1
✅ Validation : Si les pings fonctionnent, votre tunnel VPN est opérationnel.

Configuration du Firewall UFW
Sécurisation du serveur

1. Installation de UFW

bash — Installer UFW
sudo apt update
sudo apt install ufw -y

2. Configuration des règles

bash — Autoriser les ports
# SSH (port 22)
sudo ufw allow 22/tcp

# HTTPS (port 443)
sudo ufw allow 443/tcp

# WireGuard (port 51820)
sudo ufw allow 51820/udp

3. Activation de UFW

bash — Activer UFW
sudo ufw enable

# Vérifier les règles
sudo ufw status
✅ Résultat attendu : 
Status: active

To                         Action      From
--                         ------      ----
22/tcp                     ALLOW       Anywhere
443/tcp                    ALLOW       Anywhere
51820/udp                  ALLOW       Anywhere
22/tcp (v6)                ALLOW       Anywhere (v6)
443/tcp (v6)               ALLOW       Anywhere (v6)
51820/udp (v6)             ALLOW       Anywhere (v6)

6. Sauvegardes manuelles

Pour effectuer une sauvegarde manuelle :

bash — Sauvegarder les données
# Base de données
docker exec nextcloud-db mysqldump -u root -pCHANGE_ME_ROOT_PASSWORD nextcloud > /opt/nextcloud/backups/nextcloud-db-$(date +%Y-%m-%d).sql

# Fichiers
sudo tar -czvf /opt/nextcloud/backups/nextcloud-files-$(date +%Y-%m-%d).tar.gz /opt/nextcloud/app /opt/nextcloud/data
