D. Installation avec Docker
Déploiement de Nextcloud en conteneurs
ℹ️ Prérequis :
  • Serveur sous Linux (Ubuntu/Debian recommandé)
  • Docker et Docker Compose installés
  • Accès root ou sudo
  • Ports 8007 et 51820 disponibles

D.1 — Nettoyage initial (si nécessaire)

Si une installation précédente existe, commencez par tout supprimer :

bash — Nettoyage complet
# Arrêter et supprimer les conteneurs existants
docker stop nextcloud-nginx nextcloud-app nextcloud-db
docker rm nextcloud-nginx nextcloud-app nextcloud-db

# Nettoyer les images et volumes inutilisés
docker system prune -af

# Supprimer les fichiers de configuration
sudo rm -rf /opt/nextcloud

D.2 — Création de l'arborescence

bash — Créer les dossiers
sudo mkdir -p /opt/nextcloud/{app,data,db,nginx,ssl}

D.3 — Génération du certificat SSL auto-signé

bash — Générer le certificat
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /opt/nextcloud/ssl/nextcloud.key \
  -out /opt/nextcloud/ssl/nextcloud.crt \
  -subj "/CN=nextcloud"

D.4 — Fichier docker-compose.yml

Créez le fichier de configuration Docker Compose :

bash — Créer le fichier
sudo nano /opt/nextcloud/docker-compose.yml
/opt/nextcloud/docker-compose.yml
version: '3.8'

services:

  # Base de données MariaDB
  db:
    image: mariadb:10.6
    container_name: nextcloud-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: XXXXXX
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: XXXXXX
    volumes:
      - /opt/nextcloud/db:/var/lib/mysql

  # Application Nextcloud
  nextcloud:
    image: nextcloud:28-fpm
    container_name: nextcloud-app
    restart: always
    environment:
      MYSQL_HOST: db
      MYSQL_DATABASE: nextcloud
      MYSQL_USER: nextcloud
      MYSQL_PASSWORD: XXXXXX
    volumes:
      - /opt/nextcloud/app:/var/www/html
      - /opt/nextcloud/data:/var/www/html/data
    depends_on:
      - db

  # Serveur Web Nginx
  nginx:
    image: nginx:latest
    container_name: nextcloud-nginx
    restart: always
    ports:
      - "8007:8007"
    volumes:
      - /opt/nextcloud/nginx/nextcloud.conf:/etc/nginx/conf.d/default.conf
      - /opt/nextcloud/ssl:/etc/nginx/ssl
      - /opt/nextcloud/app:/var/www/html
    depends_on:
      - nextcloud
ℹ️ Note : Remplacez XXXXXX par vos mots de passe réels dans un environnement sécurisé.

D.5 — Configuration Nginx

bash — Créer le fichier de config Nginx
sudo nano /opt/nextcloud/nginx/nextcloud.conf
/opt/nextcloud/nginx/nextcloud.conf
server {
    listen 8007 ssl;
    server_name _;

    # Certificat SSL
    ssl_certificate /etc/nginx/ssl/nextcloud.crt;
    ssl_certificate_key /etc/nginx/ssl/nextcloud.key;

    # Racine du site
    root /var/www/html;
    index index.php;

    # Taille max des fichiers
    client_max_body_size 2048M;

    # Gestion des URLs
    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    # Gestion de PHP
    location ~ \.php$ {
        fastcgi_pass nextcloud-app:9000;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}

D.6 — Lancement des conteneurs

bash — Démarrer Nextcloud
cd /opt/nextcloud
docker compose up -d
✅ Accès à Nextcloud : https://IP_DU_SERVEUR:8007

E. Configuration VPN WireGuard
Connexion sécurisée entre Nextcloud et le NAS

E.1 — Installation de WireGuard

bash — Installer WireGuard
sudo apt update
sudo apt install wireguard -y

E.2 — Génération des clés

Sur le serveur Nextcloud :

bash — Générer les clés pour Nextcloud
wg genkey | sudo tee /etc/wireguard/privatekey | wg pubkey | sudo tee /etc/wireguard/publickey

Afficher les clés générées :

bash — Afficher les clés
# Clé privée Nextcloud (À PROTÉGER)
sudo cat /etc/wireguard/privatekey

# Clé publique Nextcloud
sudo cat /etc/wireguard/publickey
⚠️ Attention : Les clés privées doivent être stockées de manière sécurisée. Ne les partagez jamais en clair.

Sur le NAS Synology :

bash — Générer les clés pour le NAS
wg genkey | tee nas_privatekey | wg pubkey > nas_publickey

E.3 — Configuration WireGuard sur Nextcloud

bash — Configurer WireGuard
sudo nano /etc/wireguard/wg0.conf
/etc/wireguard/wg0.conf
[Interface]
Address = 10.10.10.1/24
PrivateKey = CLÉ_PRIVÉE_NEXTCLOUD
ListenPort = 51820

[Peer]
PublicKey = CLÉ_PUBLIQUE_NAS
AllowedIPs = 10.10.10.2/32

Activer WireGuard :

bash — Activer le VPN
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0

E.4 — Configuration WireGuard sur le NAS

Sur le NAS Synology, créez un fichier de configuration WireGuard avec :

Configuration NAS
[Interface]
Address = 10.10.10.2/24
PrivateKey = CLÉ_PRIVÉE_NAS

[Peer]
PublicKey = CLÉ_PUBLIQUE_NEXTCLOUD
Endpoint = IP_PUBLIC_NEXTCLOUD:51820
AllowedIPs = 10.10.10.1/32
PersistentKeepalive = 25
ℹ️ Note : Remplacez CLÉ_PRIVÉE_NEXTCLOUD, CLÉ_PUBLIQUE_NAS, etc. par vos valeurs réelles dans un environnement sécurisé.

E.5 — Test de la connexion VPN

Depuis Nextcloud :

bash — Tester la connexion vers le NAS
ping 10.10.10.2

Depuis le NAS :

bash — Tester la connexion vers Nextcloud
ping 10.10.10.1

G. Configuration du Firewall UFW
Sécurisation du serveur Nextcloud

G.1 — Installation de UFW

bash — Installer UFW
sudo apt update
sudo apt install ufw -y

G.2 — Configuration des règles

bash — Autoriser les ports nécessaires
# Autoriser SSH (port 2222)
sudo ufw allow 2222/tcp

# Autoriser Nextcloud (port 8007)
sudo ufw allow 8007/tcp

# Autoriser WireGuard (port 51820)
sudo ufw allow 51820/udp

G.3 — Activation de UFW

bash — Activer le firewall
sudo ufw enable

# Vérifier les règles
sudo ufw status
