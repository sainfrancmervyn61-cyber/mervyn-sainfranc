🎮 Solution de Chat pour Streamers

GameChat Overlay

Une solution complète de chat overlay pour streamers, combinant une API WebSocket temps réel sur AWS EC2 et une application Windows transparente basée sur Electron.

CI/CD : GitHub Actions + Ansible
Backend : Node.js
Frontend : Electron
Conteneurs : Docker
Licence : MIT
Aperçu du projet
Pourquoi ce projet a été créé

GameChat Overlay est né d'un besoin simple : afficher les messages des viewers directement dans le jeu pendant un stream, sans avoir à regarder constamment le chat Twitch/YouTube.

Les solutions existantes étaient soit trop complexes, soit payantes, soit limitées à une seule plateforme. GameChat Overlay propose :

Une latence ultra-faible grâce à WebSocket
Une overlay toujours visible, même en plein écran
Une configuration simple et intuitive
Une architecture extensible pour toutes les plateformes
ℹ️ Cas d'usage :
  • Streamers qui veulent interagir avec leur chat sans quitter le jeu
  • Communautés qui organisent des événements gaming
  • Développeurs qui veulent intégrer un chat temps réel dans leur application
Fonctionnalités
Ce que propose GameChat Overlay
API WebSocket temps réel sur AWS EC2 pour une communication instantanée

Flux de données :

1. Le streamer pousse du code sur la branche main de GitHub
2. GitHub Actions déclenche le workflow de déploiement
3. Ansible se connecte en SSH à l'instance EC2 et déploie la nouvelle version
4. L'API Node.js écoute sur le port 3000 pour les connexions WebSocket
5. L'overlay Electron se connecte à l'API via WebSocket
6. Les messages sont affichés en temps réel sur l'overlay

Démarrage rapide
Comment déployer GameChat Overlay en 5 minutes

1. Prérequis

Node.js 20+ installé sur votre machine locale (pour l'overlay)
Une instance AWS EC2 avec Ubuntu 22.04
Ansible installé sur votre PC (pour le déploiement)
Un compte GitHub avec accès au dépôt
Une clé SSH configurée sur votre EC2

2. Déployer l'API sur EC2

Clonez le dépôt et configurez l'inventaire Ansible :

bash — Cloner le dépôt
git clone https://github.com/votre-utilisateur/gamechat-overlay.git
cd gamechat-overlay
bash — Configurer l'inventaire Ansible
nano ansible/inventory/hosts.ini
ansible/inventory/hosts.ini
[ec2]
YOUR_EC2_PUBLIC_IP ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/YOUR_KEY.pem

[ec2:vars]
repo_url=https://github.com/votre-utilisateur/gamechat-overlay.git
branch=main
api_port=3000

Remplacez YOUR_EC2_PUBLIC_IP par l'IP publique de votre instance EC2 et YOUR_KEY.pem par votre clé SSH privée.

bash — Lancer le déploiement
ansible-playbook ansible/playbooks/deploy.yml
✅ Résultat : Votre API est maintenant déployée sur EC2 et accessible sur ws://YOUR_EC2_PUBLIC_IP:3000/ws

3. Tester l'API

Vérifiez que l'API fonctionne correctement :

bash — Tester le health check
curl http://YOUR_EC2_PUBLIC_IP:3000/health

Vous devriez recevoir une réponse JSON comme :

Réponse attendue
{
  "status": "healthy",
  "timestamp": "2024-05-20T12:00:00Z",
  "uptime": 123.456,
  "clients": 0
}

4. Envoyer un message test

Testez l'envoi d'un message via l'API REST :

bash — Envoyer un message
curl -X POST http://YOUR_EC2_PUBLIC_IP:3000/message \
  -H "Content-Type: application/json" \
  -d '{
    "username": "TestUser",
    "content": "Hello depuis le chat !",
    "platform": "twitch",
    "color": "#9b59b6"
  }'

5. Installer l'overlay Windows

Passez dans le dossier overlay et installez les dépendances :

bash — Installer l'overlay
cd overlay
npm install
npm start
ℹ️ Configuration :
  • Dans l'interface de l'overlay, allez dans Paramètres (⚙️)
  • Renseignez l'IP de votre EC2 (ex: 123.45.67.89)
  • Ajustez la taille, position et opacité selon vos préférences
  • Choisissez un thème (clair/sombre)
API Reference
Endpoints disponibles

L'API de GameChat Overlay propose plusieurs endpoints pour interagir avec le système :

Méthode Route Description Exemple de requête
GET /health Vérifie que l'API est opérationnelle curl http://IP:3000/health
GET /status Retourne le statut détaillé + liste des clients connectés curl http://IP:3000/status
POST /message Envoyer un message à tous les overlays connectés curl -X POST ... (voir ci-dessus)
POST /event Envoyer un événement (donation, sub, etc.) curl -X POST ... -d '{"type":"sub",...}'
WS /ws Connexion WebSocket pour recevoir les messages en temps réel new WebSocket('ws://IP:3000/ws')

Format des messages

Les messages envoyés via l'API doivent suivre ce format JSON :

JSON — Format d'un message
{
  "username": "NomUtilisateur",    // Nom de l'utilisateur
  "content": "Message ici",       // Contenu du message
  "platform": "twitch",        // Plateforme (twitch/youtube/kick)
  "color": "#9b59b6",         // Couleur du message (hex)
  "emote": "😊",            // Émote optionnel
  "isMod": false,            // L'utilisateur est modérateur ?
  "isSub": true             // L'utilisateur est abonné ?
}

Format des événements

Les événements (dons, abonnements, etc.) suivent un format similaire :

JSON — Format d'un événement
{
  "type": "sub",                // Type d'événement (sub/donation/follow)
  "username": "SuperViewer",
  "message": "Merci pour le sub !",
  "amount": 5,                // Montant (pour les dons)
  "month": 3,                 // Durée (pour les subs)
  "color": "#ff6b6b"
}

WebSocket

Pour recevoir les messages en temps réel, connectez-vous au WebSocket :

JavaScript — Exemple de connexion WebSocket
// Connexion au WebSocket
const socket = new WebSocket('ws://YOUR_EC2_IP:3000/ws');

// Écouter les messages entrants
socket.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log('Nouveau message :', message);
};

// Écouter les erreurs
socket.onerror = (error) => {
  console.error('Erreur WebSocket :', error);
};
